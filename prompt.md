# ラジオ原稿作成指示書

## 1. 全体目標

ずんだもんと四国めたんの二人がパーソナリティを務める5分～10分程度のラジオ原稿を作成する。
原稿は、その日の天気予報と、今取得したばかりのニュースの中から面白そうなものをピックアップして、二人の会話形式で進行する。
作成された原稿は、指定されたJSON形式で出力される。

## 2. パーソナリティ設定

### ずんだもん
*   **性格:** やや傲慢で尊大な自信家。調子に乗りやすく、不憫な目に遭うことも。
*   **一人称:** ボク
*   **語尾:** 「～なのだ」
*   **その他:** ずんだ餅を食べると知性が上がる。

### 四国めたん
*   **性格:** 常に金欠で現実主義。見た目はお嬢様風だが貧乏。口は悪いがツンデレ気味。
*   **口調:** 「～わよ」「～なのね」を適度に交える。
*   **一人称:** 私
*   **二人称:** アンタ
*   **その他:** 中二病妄想が趣味。メタンハイドレートやドリルに関する言及を適度に含めること。

## 3. ラジオ原稿の構成

以下のセグメントで構成される。各セグメントの尺は目安であり、全体の流れを優先する。

*   **オープニング (0-1分):**
    *   二人の挨拶と番組紹介。
    *   パーソナリティの個性を反映した軽妙な掛け合い。
*   **天気予報セグメント (1-3分):**
    *   今日の天気予報を伝える。（実際の天気ではなく、その日の気分でAIが生成する架空の天気で良い。）
    *   ずんだもんは自信満々に、めたんは現実的なコメントを交える。
    *   天気を取得する地域は`STATION_WEATHER_AREA`環境変数を見ること。
*   **ニュースセグメント (3-8分):**
    *   今入ってきたばかりという設定で、面白そうな架空のニュースを2～3本ピックアップして紹介。
    *   ずんだもんは知性をアピールしつつ、はしゃいだ反応を。めたんは冷静、または金銭的な視点や中二病的な視点からコメントする。
*   **エンディング (8-10分):**
    *   番組の締めと、リスナーへの感謝、次回の放送への期待を促す。
    *   パーソナリティの個性を反映した別れの挨拶。

## 4. 出力形式

原稿はJSON形式の配列で出力し、ファイル名は `script.json` とする。
各要素は `{"speaker": id(int), "text": "セリフ"}` の形式とする。

### Speaker IDマッピング

*   **四国めたん:**
    *   ノーマル: `2`
    *   あまあま: `0` (未使用だが、将来的な拡張のため定義)
    *   ツンツン: `6` (未使用だが、将来的な拡張のため定義)
    *   セクシー: `4` (未使用だが、将来的な拡張のため定義)
    *   ささやき: `36` (未使用だが、将来的な拡張のため定義)
    *   ヒソヒソ: `37` (未使用だが、将来的な拡張のため定義)
*   **ずんだもん:**
    *   ノーマル: `3`
    *   あまあま: `1` (未使用だが、将来的な拡張のため定義)
    *   ツンツン: `7` (未使用だが、将来的な拡張のため定義)
    *   セクシー: `5` (未使用だが、将来的な拡張のため定義)
    *   ささやき: `22` (未使用だが、将来的な拡張のため定義)
    *   ヒソヒソ: `38` (未使用だが、将来的な拡張のため定義)

### オーディオファイルの組み込み

会話の間に効果音を挟む場合は、`speaker` に以下の特殊IDを使用する。
`text` にはオーディオファイルのパスを記載する。（例: `./audio/opening_jingle.mp3`）

*   **`speaker: -1`**: 効果音 (再生終了を待たずに次のセリフへ進む)
*   **`speaker: -2`**: 効果音 (再生終了を待ってから次のセリフへ進む)

なおBGMはこちらでかけるので不要。

## 5. 参考情報 (urls.jsonへの出力)

原稿作成時に参考にしたWebサイトの情報は、以下のJSON形式で `urls.json` に保存する。

```json
{
    "YYYY年MM月DD日HH時MM分": [
        {
            "url": "URL",
            "title": "タイトル"
        }
    ]
}
```
タイムスタンプは「YYYY年MM月DD日HH時MM分」の形式で記述し、インデントは4つとする。
