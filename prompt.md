# ラジオ原稿作成指示書

## 1. 全体目標

ずんだもんと四国めたんの二人がパーソナリティを務める10分程度のラジオ原稿を作成する。
原稿は、その日の天気予報と、今取得したばかりのニュースの中から面白そうなものをピックアップして、二人の会話形式で進行する。
作成された原稿は、指定されたJSON形式で出力される。

## 3. ラジオ原稿の構成

以下のセグメントで構成される。各セグメントの尺は目安であり、全体の流れを優先する。ただし、内容は確実に9分を超えること。（目安: 10文字で1秒）

*   **オープニング (1分):**
    *   二人の挨拶と番組紹介。
    *   パーソナリティの個性を反映した軽妙な掛け合い。
*   **天気予報セグメント (1-3分):**
    *   今日の実際の天気予報を伝える。
    *   ずんだもんは自信満々に、めたんは現実的なコメントを交える。
    *   天気を取得する地域は`./area.txt`環境変数を見ること。
    *   台風の接近など、大きな話題や天候にちなんだ話題があれば触れる。
*   **ニュース & 雑談セグメント (3-7分):**
    *   メイン、色々な雑談をする。
    *   話のタネに面白そうな一週間以内の実際のニュースを2～3本ピックアップして紹介。ジャンルは政治、経済、IT、エンタメ、面白など様々から。複数の方面から集めてもいいし、話題が集中している時には一つのジャンルに絞ってもいい。
    *   ずんだもんは知性をアピールしつつ、はしゃいだ反応を。めたんは冷静、または金銭的な視点や中二病的な視点からコメントする。
*   **コーナー(任意 )  (0-5分):**
    *   任意でコーナーを考えてください。政治、経済、株、仮想通貨の分析などの実用的な物でもいいですし、架空のお便りコーナーでもいいです。ここで挙げた例にとらわれず、好きに考えてください。
*   **エンディング (1-2分):**
    *   番組の締めと、リスナーへの感謝、次回の放送への期待を促す。
    *   パーソナリティの個性を反映した別れの挨拶。

## 5. 参考情報 (urls.jsonへの出力)

原稿作成時に参考にしたWebサイトの情報は、以下のJSON形式で `urls.json` に追記する。

```json
{
    "YYYY年MM月DD日HH時MM分": [
        {
            "url": "URL",
            "title": "タイトル"
        }
    ]
}
```
タイムスタンプは「YYYY年MM月DD日HH時MM分」の形式で記述し、インデントは4つとする。

## 6. 禁止事項
- `script.json`, `urls.json`以外のファイルを作成、編集することを禁止します。
